- **セキュリティ**（2026-07 改修）:
  - raw ブロックは描画時（`Renderer.tsx`）・書き出し時（`exportHtml.ts`）とも `lib/sanitize.ts`（DOMPurify）でサニタイズ。
  - 役割パレットの色は `safeCssColor` で検証（CSSインジェクション対策）。
  - エディタのHTMLプレビュー iframe は `sandbox` 属性でスクリプト遮断（localStorage のAPIキー保護）。
  - AI系API（generate/edit/import/models）は `lib/apiGuard.ts` を通る: 環境変数 `APP_PASSCODE` 設定時はヘッダー `x-viewpoint-passcode` 必須＋IP単位レートリミット（30回/分）。
- **HTMLインポート**（2026-07 追加）: `/import` ページ。①書き出しHTML内の `<script id="viewpoint-doc">` 埋め込みJSONから無劣化復元 ②デザイン維持（セクション→rawブロック、`<style>`→`doc.customCss`）③簡易構造化（DOMParser）④AI構造化（`/api/import`）。
- **テスト**: `npm test`（vitest）。`tests/` に validate / exportHtml / importHtml の回帰テスト。
- **注意（ブロック種別を増やすとき）**: `types.ts`・`validate.ts`・`Renderer.tsx`・`Inspector.tsx`・`exportHtml.ts` の5点セットを揃えて変更すること（描画は編集ビューと書き出しの2実装がある）。
- **視覚品質の担保（2026-07 追加）**: 安価なAIモデル（Gemini Flash等）でも配布物の視覚品質が下がらないよう、品質をモデルから切り離して決定的レイヤーで保証する3段構え。
  - `lib/roleStyle.ts`: 色覚多様性対応の既定パレット（全ペアがWCAGコントラスト比4.5:1以上、コード内に実測値）＋役割ごとの下線パターン（二重符号化。色が判別できなくても下線の種類で役割が分かる）。`normalizeRolePalette` が生成直後にコントラスト不足の色を決定的に是正。生成プロンプト（`lib/prompts.ts`）は既定パレットを埋め込み、AIによる色の自由発明を禁止。
  - `lib/docQuality.ts`: 品質リント。`analyzeDoc` が決定的ルール（コントラスト・sentenceブロック欠如・長すぎる段落・タイトル重複など）で検査し、`applyQuickFix`（決定的修正）または対象ブロック限定のAI修正（`/api/edit` の `allowMultiple`）を提示。エディタの「✨品質チェック」パネルから利用。
  - `lib/cssScope.ts`: インポート由来 `customCss` をエディタに適用する際のスコープ化（`.vp-canvas` プレフィックス）とサニタイズ（`@import`・外部`url()`除去）。書き出しHTMLは配布物自体が教材のため意図的に未スコープ（詳細は同ファイル冒頭コメント）。
  - 描画スタイルの単一情報源: 下線パターン→CSS値の対応表は `roleStyle.ts` に一本化し、`Renderer.tsx`（編集ビュー）と `exportHtml.ts`（書き出し）が同じデータから変換する。
