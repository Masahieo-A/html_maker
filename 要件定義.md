# 要件定義書 — 教材オーサリングアプリ（仮称: Viewpoint）

> 構造化コンポーネント方式で、英語の複雑な文構造などを「視覚的に分かる教材」として作り、
> HTMLで生徒に即共有できる Web アプリ。AI生成を出発点に、教員がノード単位で細部まで編集する。

---

## 1. 背景と目的

- 英語の複雑な文を、樹形図・色分け・注釈などで**視覚的に理解させる教材**を作りたい。
- Word/PPT より HTML が望ましい理由：生徒への**即時共有**が可能で、扱える視覚表現が広い。
- AIの一発出しHTMLは、教員の伝えたいメッセージ・こだわりを細部まで表現できない。
  → **AI生成を下書きとし、そこから教員がGUIで細部を作り込める**仕組みが必要。
- 教員はコード（HTML/CSS）に触れない。すべて画面上の直接操作で完結する。

## 2. 中核コンセプト：構造化コンポーネント方式

**見た目（HTML）を直接編集しない。** 教材を「意味の単位（コンポーネント＝ノード）」の集合データとして持ち、
描画はそのデータから自動生成する。編集は常に**データに対して**行う。

この方式を採る理由：

- どの粒度でも「選択して編集」が等しく成立する（単語1語、樹形図の枝1つまで）。
  → **どこまでノードに分解するかが、そのまま編集可能な粒度になる。**
- AIによる部分編集が、生HTMLの切り貼りではなく**データ単位のやり取り**になるため、
  レイアウトが崩れず、undo/履歴が自明に実装できる。
- 同一データから「編集用の描画」と「生徒配布用の静的HTML」を両方生成できる。

> 制約の対価：AIの出力を自由HTMLではなくコンポーネント語彙に「通す」必要がある。
> 語彙でカバーしきれない例外用に `raw`（生HTML）ブロックを逃げ道として用意する。

## 3. ファイル形式（2種）

| 形式 | 役割 | 例え |
|---|---|---|
| `*.json` | 編集・保存・再編集される**ソース（信頼できる唯一の実体）** | PowerPoint の `.pptx` |
| `*.html` | 生徒へ配布する**書き出し成果物**（編集機能なし・静的） | 書き出した PDF |

- `.json` は Supabase に永続化。`.html` は同じ JSON から都度生成し、ダウンロード or 共有URLで配布。
- 書き出しHTMLは**自己完結（CSSインライン）・レスポンシブ**であること。

## 4. データモデル（最重要・設計の背骨）

TypeScript 型で契約を定義する。これが AI 生成の出力スキーマ、描画、編集すべての基準になる。

```ts
type LessonDoc = {
  id: string;
  title: string;
  version: number;
  // 役割名 → 色 のパレット。色分けは "意味（働き）" を表すので doc 単位で定義し再利用する
  rolePalette: Record<string, { label: string; color: string; bg: string }>;
  blocks: Block[];
};

type Block =
  | HeadingBlock
  | ParagraphBlock
  | SentenceBlock
  | TreeBlock
  | NoteBlock
  | ImageBlock
  | RawHtmlBlock;     // 逃げ道（語彙で表せない例外用）

type HeadingBlock   = { id: string; type: "heading"; level: 1|2|3; text: string };
type ParagraphBlock = { id: string; type: "paragraph"; text: string };

// 文：1語ずつをノードとして持ち、役割（=色）を割り当てる
type SentenceBlock  = { id: string; type: "sentence"; tokens: Token[] };
type Token          = { id: string; text: string; role: string | null }; // role は rolePalette のキー

// 樹形図：root + 枝。枝は入れ子可能（実際の係り受け・多層構造に対応）
type TreeBlock      = { id: string; type: "tree"; root: string; branches: Branch[] };
type Branch         = { id: string; role: string; value: string; children?: Branch[] };

// 注釈ボックス（着眼点・ヒントなど）
type NoteBlock      = { id: string; type: "note"; label: string; body: string;
                        variant?: "point" | "tip" | "warning" };

type ImageBlock     = { id: string; type: "image"; src: string; alt: string };
type RawHtmlBlock   = { id: string; type: "raw"; html: string };
```

設計上の決まり事：

- すべてのノードに一意の `id` を持たせる（選択・部分編集・undo の対象単位）。
- 色は装飾でなく**意味**。必ず `role`（rolePalette のキー）経由で付け、ハードコードしない。
- 樹形図の**線・配置はAIに描かせない**。`TreeBlock` の構造データを、決定的なレイアウタ
  （dagre / elk / d3-hierarchy 等）で描画する。AIは木の**構造**だけを出力する。

JSON 例（最小）：

```json
{
  "id": "lesson3", "title": "and / but / or を見抜く", "version": 1,
  "rolePalette": {
    "and": { "label": "and（並列）", "color": "#0d9488", "bg": "#d7f3ef" },
    "or":  { "label": "or（選択）",  "color": "#c2740a", "bg": "#fbeccd" }
  },
  "blocks": [
    { "id": "b1", "type": "heading", "level": 2, "text": "and / but / or を見抜く" },
    { "id": "b2", "type": "sentence", "tokens": [
      { "id": "t1", "text": "desire", "role": "and" },
      { "id": "t2", "text": "and", "role": "and" },
      { "id": "t3", "text": "determination", "role": "and" }
    ]},
    { "id": "b3", "type": "tree", "root": "S（文の骨組み）", "branches": [
      { "id": "br1", "role": "主部 (S)", "value": "desire and determination" },
      { "id": "br2", "role": "動詞 (V)", "value": "exceed" }
    ]},
    { "id": "b4", "type": "note", "label": "着眼点",
      "body": "and は主語の名詞2つを繋ぐ。前後が同じ品詞かを見る。", "variant": "point" }
  ]
}
```

## 5. 主要機能

### 5.1 AI生成（教材の下書き作成）

- 入力：本文・着眼点などの**テキスト**＋（任意）**手書きスケッチの写真**。
- 写真入力は補完用。`<input type="file" accept="image/*" capture="environment">` で
  スマホはカメラ起動／PCはファイル選択。撮影画像を base64 でマルチモーダルモデルへ渡す。
- 出力：**`LessonDoc` スキーマに厳密に従う JSON のみ**（前置き・Markdown 装飾なし）。
- 受信側で安全にパースし、不正なら 1 回まで自動リトライ（修正指示付き）。

### 5.2 手動編集（ノード単位の WYSIWYG）

- 描画済みの教材ビュー上で、**任意のノードをタップ → インスペクタでプロパティ編集**。
- 単語1語、樹形図の枝の「役割」「中身」まで個別に選択・編集できること（＝すべて編集対象）。
- 編集は `LessonDoc` を書き換えるだけ。ビューと書き出しHTMLは自動で追随。

### 5.3 範囲指定 AI 編集

- ビュー上でノードを選択 → 右ペインに自然言語で改善指示 → 該当ノードの**データのみ**を
  AIに渡し、**更新後のノードデータ**を受け取って差し替える。
- スコープは選択ノードに限定。返却も同じスキーマに制約し、整合性を保つ。
- すべてデータ操作なので **undo/redo（履歴スタック）** が自明に効く。

### 5.4 保存・読み込み

- `LessonDoc`（JSON）を Supabase に保存。一覧・複製・再編集が可能。
- 教員アカウントごとに教材を管理（認証は Supabase Auth）。

### 5.5 HTML 書き出し（生徒共有）

- 同一 JSON から自己完結・レスポンシブな静的 HTML を生成。
- 配布方法：ファイルダウンロード and/or 閲覧専用の共有URL（編集UIなし）。

## 6. 画面構成

1. **教材一覧**：作成済み `LessonDoc` の一覧／新規作成／複製。
2. **生成画面**：テキスト入力＋スケッチ写真アップロード → 「AIで下書き生成」。
3. **エディタ画面**（中心）：
   - 中央：教材ビュー（データから描画。クリックでノード選択）。
   - 右（モバイルでは下）：インスペクタ（選択ノードのプロパティ編集＋範囲指定AI編集）。
   - タブ：`教材ビュー / ソース(JSON) / 書き出し(HTML)`。
   - 操作：undo/redo、保存、HTML書き出し。

## 7. 技術スタック・アーキテクチャ

> 既存資産に合わせる。下記は既定案。分離が必要なら FastAPI バックエンドも可。

- フロント：**Next.js（App Router）/ React / TypeScript**。
- 描画：データ → React コンポーネント（編集ビュー）／データ → 静的HTML文字列（書き出し）。
- 樹形図レイアウト：dagre / elk / d3-hierarchy のいずれか。
- AI：**Claude もしくは Gemini API**（BYOK 前提。モデルはコスト優先で選択可能に）。
  - 生成・部分編集は **structured output（JSONのみ返す）** をシステムプロンプトで強制。
- 永続化／認証：**Supabase**（Postgres + Auth + Storage）。スケッチ写真は Storage 保管 or 即時破棄を選択可。
- デプロイ：**Vercel**。
- 状態管理：`LessonDoc` を単一の信頼できる状態とし、編集は純粋な変換関数 ＋ 履歴スタックで実装。

## 8. 非機能要件

- **出力HTML**：自己完結・レスポンシブ・印刷可。生徒のスマホで崩れないこと。
- **編集UI**：デスクトップ／タブレット主体。閲覧（生徒側）はモバイル最適。
- **undo/redo**：データ操作ベースで全編集に対応。
- **コスト**：BYOK、モデル選択可、トークン最小化（部分編集は該当ノードのみ送信）。
- **プライバシー**：本ツールは教員オーサリング用途。生徒個人情報は扱わない設計を基本とし、
  スケッチ画像の保存可否は設定で制御。
- **アクセシビリティ**：キーボード操作・フォーカス可視・色覚配慮（色＋ラベルの二重符号化）。

## 9. スコープと開発フェーズ

**フェーズ1（MVP）**
- データモデル（型）＋描画（編集ビュー）＋ HTML 書き出し。
- AI生成（テキスト＋写真 → JSON）。
- ノード単位の手動編集（インスペクタ）。
- Supabase 保存・読み込み、一覧。

**フェーズ2**
- 範囲指定 AI 編集（選択ノード → 指示 → 差し替え）＋ undo/redo。
- コンポーネント語彙の拡充、樹形図の自動整列（多層・係り受け）。
- 共有URL（閲覧専用）。

**フェーズ3（任意）**
- 表示中教材への手書き注釈オーバーレイ（写真ではなくキャンバスで矢印・マル）→ そのままAIへ。
- 教材テンプレート、`role` パレットのプリセット集。

## 10. 未決定事項（着手前に決めると吉）

- コンポーネント語彙の最終リスト（最初は heading / paragraph / sentence / tree / note / image / raw で開始）。
- 編集粒度：文を「語」までで十分か、形態素・文字単位まで分解するか。
- `role`（役割ラベル）を固定リスト選択にするか自由入力にするか。
- 共有方式：ダウンロード中心か、共有URL中心か。
- バックエンド分離（FastAPI）の要否。

## 11. MVP 受け入れ基準

- [ ] テキスト（＋任意で写真）を入力すると、スキーマ準拠の `LessonDoc` が生成され描画される。
- [ ] ビュー上の任意ノード（見出し・段落・単語・樹形図の root と各枝・注釈）を選択し編集できる。
- [ ] 編集結果が JSON タブと HTML タブに即時反映される。
- [ ] `LessonDoc` を保存し、一覧から再編集できる。
- [ ] 自己完結・レスポンシブな HTML を書き出せ、別端末のブラウザで正しく表示される。
